4. 4視点 独立 Chrome MCP 監査 詳細

各監査は独立した人格として Claude が Chrome MCP を操作。スクリーンショット・Lighthouse・DevTools・人格シミュレーションを総合。1視点でも Fail なら Go-Live 不可。

  • §6: Polaris 25 コンポーネント実装
  • §7: 状態網羅 10×5=50 セル

  • 読込中 / エラー状態を強制発火
  • Lighthouse A11y 自動実行
  • Axe DevTools 自動監査

② Axe critical errors = 0
③ 状態網羅 50/50 セル実装確認
④ Polaris 以外のスタイル 0 件

6. ドキュメント全体マップ

3 ドキュメントは三位一体で動作します:
① ガントチャートが「いつ何をするか」のロードマップ、
② UX/UI Design Specが「何をどう作るか」の設計仕様、
③ Traceability Matrix（本ドキュメント）が「どう繋がり・どう検証するか」の橋渡し。
そしてPhase M-Auditが 4 視点で独立検証を行い、Go-Live を保証します。
